# Manual Amendments

Per `CC-PRINCIPAL-ENGINEER-MANUAL.md` §15.

## What this is

Amendments to `slyght/CC-PRINCIPAL-ENGINEER-MANUAL.md`. The manual itself is treated as semi-frozen — amendments capture the change history without rewriting the manual every time.

Per §15: when a recurring mistake or new pattern emerges, propose an amendment here, get John's approval, then update the manual + log the amendment.

## Format

`AMENDMENT-NNN-<short-name>.md` — three-digit zero-padded sequence.

## Structure

```
# AMENDMENT-NNN — <title>

Date: YYYY-MM-DD
Section affected: §N (and §M if applicable)
Driver: <recurring-incident | new-pattern | external-feedback>
Status: proposed | accepted | rejected | applied

## Trigger
<what made this amendment necessary — be specific; cite commits/sessions>

## Change
<the exact text being added/removed/modified, before+after>

## Rationale
<why this is the right amendment vs. alternatives>

## Migration
<does this invalidate any prior practice? if so, what to do about existing artifacts>
```
